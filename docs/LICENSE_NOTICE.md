## 关于"源代码"

### Tarn 不提供源代码

Tarn 是闭源专有软件，**仅以编译后的二进制形式分发**。版权人 linjoin 不向任何用户提供 Rust 源代码，不接受源代码披露请求。

### 为什么不开源？

闭源是作者的有意选择，原因包括：

1. **规则文件是智力成果**：`rules/*.txt` 耗费大量人力从全网搜集整理，开源容易被直接搬运商用
2. **系统级 root 工具的安全风险**：被恶意 fork 后植入后门、广告、挖矿代码再分发，会损害用户利益与作者声誉
3. **维护成本**：开源社区的管理（PR review、issue 分类、行为准则执行）成本高于个人开发者承受范围
4. **商业化考量**：保留商业授权通道，为未来可能的商业化留出空间

### 但我们承诺

- ✅ **二进制不含任何遥测代码**
- ✅ **不上传任何用户数据**到版权人或第三方服务器
- ✅ **不联网回传**任何信息
- ✅ **所有运行数据均在设备本地**（`/data/adb/tarn/`）
- ✅ **WebUI 默认仅绑定 127.0.0.1**，不对外暴露
- ✅ **token 鉴权**保护管理接口

如对安全性有疑虑，可用以下方式自查：
- 抓包确认无外联流量：`tcpdump -i any -w tarn.pcap` 后用 Wireshark 分析
- 检查进程网络连接：`netstat -tunp | grep tarn`
- 检查文件权限：`ls -la /data/adb/tarn/`
- 查看日志：`cat /data/adb/tarn/logs/tarn.log`

## 常见误解澄清

### ❓ "闭源 = 不安全？"

不成立。开源确实便于审计，但闭源不等于不安全。Tarn 的安全保证来自：
- 路径白名单机制（`ALLOWED_ROOTS`）
- 硬编码保护（`/system` 等永不删除）
- dry-run 预览模式
- 6 层白名单保护
- token 鉴权 + 默认本地绑定
- 无网络上传代码（可抓包验证）

### ❓ "我不能 fork 这个项目？"

正确。**你不能 fork、修改、再分发 Tarn**。但你可以：
- 在你本地修改自用（不分发）
- 通过 Issue 提交 Bug 报告和功能建议
- 贡献清理规则（提交到官方规则库，经审核后合并）

### ❓ "我能把 Tarn 放进我的 ROM 吗？"

**不能**，除非获得商业授权。TPL-2.0 明确禁止再分发，包括打包入 ROM 和刷机包。请引导你的用户从 [官方 Releases](https://github.com/linjoin/Tarn/releases) 下载。

### ❓ "我能反编译研究一下吗？"

**不能**。TPL-2.0 明确禁止逆向工程。如你有学术研究需求，可通过 Issue 联系版权人，版权人可视情况提供有限的技术说明（不提供源代码）。
