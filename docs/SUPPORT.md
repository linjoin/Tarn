## 🔍 搜索已有 Issue

提交新问题前，请先搜索 [已有 Issue](https://github.com/linjoin/Tarn/issues?q=is%3Aissue)，避免重复。

---

## 🐛 报告 Bug

使用 [Bug Report 模板](https://github.com/linjoin/Tarn/issues/new?template=bug_report.md) 提交，请提供：

- 设备型号、Android 版本、Root 框架版本
- Tarn 版本（`tarn --version`）
- 复现步骤
- `tarn doctor` 输出
- 相关日志（`tail -100 /data/adb/tarn/logs/tarn.log`）

---

## 🔒 安全漏洞

**请勿通过公开 Issue 报告安全漏洞！** 详见 [SECURITY.md](./SECURITY.md)，通过私密渠道报告。

## 💼 商业授权

如需将 Tarn 用于商业目的，通过 GitHub Issue 联系版权人。

---

## ⏰ 响应时间

Tarn 由个人开发者维护，响应时间如下：

| 类型 | 预期响应 |
|---|---|
| 安全漏洞 | 48 小时内确认 |
| Bug 报告 | 3-7 天内回复 |
| 功能建议 | 1-2 周内回复 |
| 规则贡献 | 1-2 周内审核 |
| 商业授权 | 3-5 个工作日 |

请耐心等待，不要催促。维护者会在有空时处理。

---

## 🚫 不予处理的情况

以下情况 Issue 会被直接关闭：

- ❌ 要求公开源代码
- ❌ 质疑闭源决策
- ❌ 喷发泄愤、人身攻击
- ❌ 与 Tarn 无关的问题
- ❌ 重复 Issue（未搜索已有）
- ❌ 缺少必要信息（不提供环境、日志）
- ❌ 在公开 Issue 报告安全漏洞
